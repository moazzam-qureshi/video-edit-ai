# Experiment 12 — FFmpeg EDL translator + render

**Phase:** Phase 4
**Status:** done
**Verdict:** ✅ **PASS** on the structural-render gate; **zoom/speed_up
deferred** by design to keep this experiment scoped.
**Date run:** 2026-05-17 (UTC)
**Run by:** claude (Opus 4.7)
**Host:** moazzam-vps (Ryzen 5 3600, CPU-only, ffmpeg 6.1.1)

## Goal

Take an EDL JSON produced by the Phase 3 brain and translate it into
an ffmpeg command that produces an edited mp4. This experiment focuses
on the *structural* edits — silence removal + segment concatenation —
because that's the longest path through ffmpeg's filter graph and the
one most prone to going wrong.

Captions are deferred to Exp 13. Zoom and speed_up are tagged as
"skipped" so the EDL is still consumable end-to-end.

## Gate

- An EDL produced by Phase 3 renders successfully to mp4.
- **Output duration ≈ input_duration - sum(remove_silence)** (delta ≤
  0.5 s on a 5-min clip).
- **Render RTF ≥ 0.5×** on this CPU (audit predicted 1–2× for libx264
  medium on 1080p; we use `fast` preset on 360p, expect more).
- No FFmpeg errors.

## What we ran

- **Clip:** `samples/raw/raw_5min.mp4` (300.03 s, 640×360@30fps,
  h264 + aac).
- **EDL:** `outputs/10_brain_5min/multipass_edl.json` — the multi-pass
  brain output for the 5-min clip (153 edits across 5 types: cut 9,

  clip; the Exp 10 happy-path 66-edit EDL recorded in Exp 10's
  metrics.json had been overwritten during Phase 3 iteration and
  isn't reproducible without prompt-engineering work outside this
  experiment's scope.
- **Render params:** libx264 `preset=fast`, `crf=23`, AAC audio @
  128 kbps, `+faststart`.
- **Command:**
  ```bash
  python experiments/12_ffmpeg_edl/run.py \
      --edl outputs/10_brain_5min/multipass_edl.json \
      --clip samples/raw/raw_5min.mp4 \
      --preset fast
  ```

### Key path / what made it work

Two real ffmpeg pitfalls hit during this experiment:

1. **`-filter_complex` argv length limit.** A 49-range EDL produces a
   ~6 KB filter graph as a single argv string. ffmpeg sometimes
   rejected this with "Error parsing global options: Invalid
   argument" before even reading the filter. **Fix:** write the
   filter to a file and use `-filter_complex_script /path`. Filter
   strings ≥ ~4 KB should use the file form, full stop. Worth pinning
   in the eventual product's render code.

2. **`concat` filter requires interleaved input streams.** My first
   working filter laid out streams as `[v0]...[vN][a0]...[aN]` — all
   videos then all audios — and `concat=n=N:v=1:a=1` rejected it with
   "Media type mismatch between … filter output pad 0 (video) and …
   filter input pad 1 (audio)". The fix is interleaving:
   `[v0][a0][v1][a1]...[vN][aN]`. concat consumes one of each per
   segment, in order.

Both pitfalls are exactly the kind of thing the product code needs to
get right on day one; they don't surface from API docs in any obvious
way.

### Memory cost

Peak RSS of **3.1 GB** during render is significant. ffmpeg buffers
each concat segment in memory simultaneously — and with 48 segments
over a 5-min input it adds up. Scaling implications:

- A 30-min input at this segment density → ~290 segments → projected
  RSS ~18 GB.
- A 2-hour input → ~1,160 segments → projected RSS likely OOMs on
  this 64 GB box if also running a Whisper worker (8.5 GB).

**Mitigation for the eventual product:** for longer clips, do the
concat in chunks (e.g., 30 segments at a time, intermediate mp4 files,
then a final remux). This trades a small wall-clock penalty for bounded
memory. Flag in FINDINGS.

## Open questions / follow-ups

- **Zoom_in deferred.** Applying `zoompan` to specific subranges in the
  filter graph requires splitting kept ranges into "natural" and
  "zoomed" pieces and stitching them back. Worth a small Exp 12b — or
  fold into Exp 14 (full pipeline).
- **Speed_up deferred.** Same story — `setpts` on the kept video and
  `atempo` on the kept audio for the speed-up ranges. Splits get
  complex; defer.
- **SFX overlay deferred.** No SFX library in the repo. Product will
  ship with a small `assets/sfx/` directory of stock whoosh/click/
  swish sounds and overlay them with `amix` + `adelay`.
- **Memory at scale.** Per the analysis above, the eventual product
  must implement chunked concat for clips with > ~50 kept ranges.
- **Filter-complex-script is mandatory.** Permanent lesson; should
  be the default in product code.
